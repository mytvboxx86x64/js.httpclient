Client Setup
Shared instance used by all panels. Changes apply immediately.
onError(callback)
Intercept errors before they throw. Call retry() to re-run the original request. Only fires once per request — retries that fail are thrown directly.
The callback is set on the shared client instance. Clear it after testing to avoid affecting other panels.
get(endpoint, params?, options?)
Query params are serialized via URLSearchParams and appended to the URL.
post(endpoint, data?, options?)
Sends a JSON body. Content-Type: application/json.
put(endpoint, data?, options?)
Full replacement update.
patch(endpoint, data?, options?)
Partial update — only the fields provided are changed.
delete(endpoint, data?, options?)
Optional JSON body for DELETE-with-body scenarios.
upload(endpoint, formData, options?)
Multipart POST. Content-Type is omitted so the browser sets the boundary automatically.
jsonplaceholder does not accept real uploads — this demonstrates FormData construction and outgoing headers.
download(endpoint, params?, filename?, options?)
GET that receives a Blob and triggers a browser file-save. Filename from Content-Disposition or explicit override.
File will be saved to your Downloads folder.
namespace(prefix)
Returns a proxy that prepends a path segment to every HTTP call.
