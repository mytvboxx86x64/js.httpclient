Client Setup

Configure the shared HttpClient instance used by all panels below.

get(endpoint, params?, options?)

Performs a GET request. Query params are serialized via URLSearchParams.

Leave empty for no query params.

post(endpoint, data?, options?)

Sends JSON body. Content-Type: application/json.

put(endpoint, data?, options?)

Full replacement update. Sends JSON body.

patch(endpoint, data?, options?)

Partial update. Only the fields provided are changed.

delete(endpoint, data?, options?)

Deletes a resource. Optional JSON body for DELETE-with-body scenarios.

upload(endpoint, formData, options?)

Multipart POST. Content-Type is omitted so the browser sets the boundary automatically.

Note: jsonplaceholder.typicode.com does not accept real uploads — this demonstrates the FormData construction and shows the outgoing request headers. Use your own endpoint for actual uploads.

download(endpoint, params?, filename?, options?)

GET that receives a Blob and triggers a browser file-save. Filename can be explicit or inferred from Content-Disposition.

Saving will be triggered by the browser. Check your Downloads folder.

namespace(prefix)

Returns a Proxy that prepends a path segment to every HTTP call. All methods (get, post, put, …) are available on the returned object.
